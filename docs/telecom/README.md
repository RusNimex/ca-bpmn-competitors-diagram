Цель: Оптимизировать расходы на отправку смс для авторизации пользователей 

Задача: предложить альтернативные способы авторизации для пользователей в веб и мобильном приложении 

Контекст: На текущий момент авторизация происходит с помощью кода из СМС, отправленного на номер телефона пользователя. Основная аудитория продукта, использующая коды по СМС для авторизации — водители такси, которые могут не очень хорошо понимать, как работает интернет. Нужно максимально сократить количество отправляемых СМС на номера телефонов для авторизации. Бизнес настаивает на использовании только Telegram-бота для авторизации. Как шаги предпримешь для описания задачи? Какие варианты авторизации предложишь? Как они будут взаимодействовать между собой?
